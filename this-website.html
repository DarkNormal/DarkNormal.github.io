Site Breakdown

Home Page

Starting from the top, a material design navbar was added for navigation purposes. This navbar has a navigation drawer built into it's functionality, to give a native-application feeling to the website when using it on small devices. The search feature in the navbar does nothing, and is only there as a placeholder. The source for this is Material Design Lite, a framework developed by Google.
The Bootstrap grid was widely used throughout the website, which I created from scratch. I used different sized columns, as well as offsets and push/pull CSS classes to re-order the columns when the screen size is reduced. I previously used Bootstrap, but never knew about the push/pull feature so this was of great benefit. Offsetting columns also gave the website a better layout on larger screens, rather than each column sitting beside each other. Containers, Fluid containers and media queries were also used as part of Bootstrap.
The featured portfolio section is a set width after a certain screen width via a media query, and the on-hover effect of the featured portfolio is taken from the Freelancer Bootstrap template. I attempted to create a parallax-style to the page, however this didn't work very well in Safari-based browsers, so I removed it. All of the icons are provided via FontAwesome which provides over 440 icons. Nearly all of the HTML and CSS was written by myself, only taking some advice from StackOverflow or some tutorials online.

About Page

The About page features many of the same elements as the Home page, the navbar and the footer are the same for consistency. The timeline feature on this page is based on a Codepen by betul, but I applied some different transitions based on which side the tab is on. The CSS and most of the HTML for the timeline is from this source.

Portfolio Page

The default element on the portfolio page is a Bootstrap accordion element, however I changed the default action of the tabs opening and closing when another is selected. Instead, I kept the first one open and it stays open if the other is selected.

GitHub Pages

By committing all the website files to a GitHub repository, a website can be accessed using GitHub Pages. The repository must be called your_github_username.github.io, and this will allow the website to be created. This process is just as easy as using Azure, and somewhat better as it enables automatic source-control.
